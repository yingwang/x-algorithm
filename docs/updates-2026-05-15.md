# 11. 2026-05-15 更新明细

这次更新把仓库从"代码骨架"提到了"开箱可跑 + 内容理解 + 广告"完整版本。下面把每条更新对应到具体文件，并解释**为什么这条更新有意义**。

## ① 端到端推理脚本

**新增：** [`phoenix/run_pipeline.py`](https://github.com/yingwang/x-algorithm/blob/main/phoenix/run_pipeline.py)

**替换：** 原先分开的 `phoenix/run_retrieval.py` + `phoenix/run_ranker.py`（仍保留，但不推荐）。

**做了什么：**

```bash
uv run run_pipeline.py --artifacts_dir artifacts/oss-phoenix-artifacts
```

一行命令完成"召回 → 排序"，跟生产链路顺序一致。详见 [§8 本地运行 Phoenix 推理](running.md)。

**为什么有意义：** 让外部读者真的能跑通 Demo，而不是只读代码猜模型行为。

---

## ② 预训练 Mini 模型 artifact

**新增：** `phoenix/artifacts/oss-phoenix-artifacts.zip`（~3 GB · 通过 Git LFS 分发）

包含：

- 召回模型 checkpoint（Transformer + Candidate Tower）
- 排序模型 checkpoint（Transformer + 19 个动作头 + 8 个连续头）
- 用户 / 物品 / 作者三套多哈希 embedding 表（各 1M 行 × 128 维）
- 537K 条 sports 帖的预计算 candidate embedding 索引（`sports_corpus.npz`）
- 一个示例用户互动序列（`example_sequence.json`）

**为什么有意义：** 过去开源推荐系统几乎不附权重，本仓库是少数能开箱推理的。

---

## ③ Grox 内容理解服务

**新增：** [`grox/`](https://github.com/yingwang/x-algorithm/tree/main/grox)

含分类器、嵌入器、任务调度引擎，覆盖：

- 垃圾信息识别（`task_spam_detection.py`）
- PTOS 策略 / 品类（`task_safety_ptos_policy.py` / `task_safety_ptos_category.py`）
- 综合安全筛查（`task_post_safety_screen_deluxe.py`）
- 多模态 embedding（`task_multimodal_post_embedding.py` + sink）
- 摘要 → embedding 链路（`task_summarizer_for_post_embedding.py`）
- Grok-based UPA 动作打标（`task_grok_upa_action_with_labels.py`）
- ASR / 媒体 / 回复排序 / 速率限制等

详见 [§3.5 Grox](components-grox.md)。

**为什么有意义：** 把内容理解从"散落各处"抽成独立服务，让 home-mixer 通过下游 store 拿到现成的安全标签 / 品类 / embedding，无需自己重算。

---

## ④ 广告插入与品牌安全

**新增：** [`home-mixer/ads/`](https://github.com/yingwang/x-algorithm/tree/main/home-mixer/ads)

- `safe_gap_blender.rs`：广告与敏感内容的安全间隔（不在血腥/争议帖旁插广告）
- `partition_organic_blender.rs`：有机帖 + 广告的混排策略
- `util.rs`：插入位置工具

外层 `ForYouCandidatePipeline` 用 `AdsSource` 拉广告 → `BlenderSelector` 调用上面的 blender 决定插入位置。详见 [§3.1 Home Mixer](components-home-mixer.md) 的"两条嵌套 Pipeline"。

**为什么有意义：** 把广告 / 品牌安全独立成模块，与核心推荐解耦，让广告策略可以单独迭代。
